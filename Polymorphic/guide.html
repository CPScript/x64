Overview

The Polymorphic Shellcode Encoder/Decoder Framework is an enterprise-grade security research tool designed to transform standard shellcode into polymorphic variants that evade signature-based detection mechanisms. By implementing advanced encoding techniques and self-modifying code patterns, the framework provides robust protection against static analysis while maintaining reliable execution characteristics.

IMPORTANT: This tool is intended for security research, education, and legitimate penetration testing only. Always obtain proper authorization before using in any environment.

Polymorphic shellcode works by wrapping the original payload in a mutating decoder stub that:

  • Uses multiple layers of encoding with variable keys
  • Incorporates randomized junk instructions
  • Implements position-independent self-modifying code
  • Dynamically decodes the original payload at runtime

Key Features

Multi-Layer Encoding

Combines XOR, ROL (rotate left), and ADD operations with randomized keys for maximum signature diversity.

Polymorphic Code Generation

Each encoding produces a unique decoder stub with variable instruction sequences and random junk instructions.

Self-Modifying Decoder

Runtime decoding that modifies encoded shellcode in-place to reveal the original payload only during execution.

Position-Independent Execution

Uses call/pop technique for self-referencing, enabling execution from any memory location.

Integrated Testing Capabilities

Built-in functionality to validate encoded payloads in a controlled execution environment.

Extensible Architecture

Modular design supporting custom encoding layers and stub generation techniques.

Architecture

Encoding Process

The framework implements a multi-layered encoding approach:

Memory Layout

The encoded payload has a specialized memory structure with metadata and the encoded shellcode:

┌────────────────────────────┬────────────────────┬──────────────────────────────────────────────┐ │ Decoder Stub │ Metadata │ Encoded Shellcode │ ├────────────────────────────┼────────────────────┼──────────────────────────────────────────────┤ │ Self-modifying ASM code │ Byte 0: Length │ │ │ with polymorphic │ Byte 1: XOR Key │ Multi-layer encoded payload │ │ characteristics │ Byte 2: ROL Key │ (XOR + ROL + ADD) │ │ │ Byte 3: ADD Key │ │ │ │ Bytes 4-7: Reserved│ │ └────────────────────────────┴────────────────────┴──────────────────────────────────────────────┘

Decoder Execution Flow

The x64 assembly decoder stub uses an indirect call/pop technique to achieve position-independent execution:

  1. Jump to shellcode address getter
  2. Call decoder stub (pushing return address on stack)
  3. Pop address into register (RSI)
  4. Load metadata (length and keys)
  5. Decode shellcode byte-by-byte with multiple layers
  6. Execute decoded shellcode

Installation

Requirements

  • Python 3.6+
  • NASM assembler (for custom stub manipulation)
  • GCC compiler (for testing)

Setup

The framework is self-contained in a single Python file. Download and make it executable:

wget https://example.com/poly_encoder.py chmod +x poly_encoder.py

To verify installation:

./poly_encoder.py --help

Usage

Command Reference

The framework provides several commands for different operations:

Command Description Example
encode Encode shellcode with polymorphic techniques ./poly_encoder.py encode -i input.bin -o output.bin
test Test encoded shellcode execution ./poly_encoder.py test -i encoded.bin
extract-stub Extract decoder stub template ./poly_encoder.py extract-stub -o decoder_stub.asm

Encoding Options

The encode command supports several options:

Option Description
-i, --input FILE Input shellcode file (raw binary)
-x, --hex STRING Shellcode as hex string
-o, --output FILE Output file for encoded shellcode
--xor-key VALUE Specific XOR key (1-255)
--rol-key VALUE Specific ROL key (1-7)
--add-key VALUE Specific ADD key (1-255)
--junk-probability VALUE Probability of inserting junk instructions (0.0-1.0)
-v, --verbose Enable verbose output

Usage Examples

Basic Encoding

Encode a raw binary shellcode file 
./poly_encoder.py encode -i shellcode.bin -o encoded.bin
Encode shellcode from a hex string 
./poly_encoder.py encode -x "48 31 c0 50 48 89 e6 48 89 e2 48 8d 3c 02" -o encoded.bin

Advanced Encoding

Use custom keys for deterministic encoding 
./poly_encoder.py encode -i shellcode.bin -o encoded.bin --xor-key 42 --rol-key 3 --add-key 7
Maximize polymorphic characteristics with high junk probability 
./poly_encoder.py encode -i shellcode.bin -o encoded.bin --junk-probability 0.6 -v

Testing

Test encoded shellcode 
./poly_encoder.py test -i encoded.bin

Security Considerations

While the framework provides robust obfuscation techniques for evading signature-based detection, several important security considerations should be noted:

LEGAL DISCLAIMER: This tool is provided for research and authorized security testing only. Unauthorized use may violate local laws. Always obtain proper permission before testing.

Limitations

  • Behavioral Detection: The framework focuses on signature evasion but doesn't protect against behavioral analysis.
  • Heuristic Analysis: Advanced security solutions may flag the decoder stub based on heuristic patterns.
  • Size Increase: Polymorphic encoding increases payload size by approximately 2-3x.
  • Length Limitation: Current implementation supports shellcode up to 255 bytes (expandable with minimal modifications).

Best Practices

  • Regularly regenerate encodings for maximum polymorphic diversity
  • Combine with other evasion techniques for comprehensive protection
  • Test in isolated environments before deployment
  • Avoid using predictable keys or patterns
